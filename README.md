# Лабораторная работа: Параллельные вычисления

Реализация и сравнение трёх фильтров обработки изображений в четырёх вариантах параллелизма.

## 📋 Что в работе

**Фильтры:**
- Инверсия цветов
- Обнаружение границ (оператор Собеля)
- Медианный фильтр

**Реализации параллелизма:**
- Последовательная (baseline)
- OpenMP (параллелизм по ядрам CPU)
- SIMD (ARM NEON — векторизация внутри ядра)
- OpenCL (вычисления на GPU)

Итого 12 функций — каждый фильтр в каждой реализации.

## 🖥 Среда выполнения

- **Платформа:** macOS, Apple Silicon M3 Pro
- **CPU:** 11 ядер ARM64
- **GPU:** Apple M3 Pro, 18 compute units
- **Компилятор:** Apple Clang 15.x (LLVM)
- **Стандарт:** C++17, оптимизация `-O3`

> Задание ориентировано на x86_64 + Visual Studio. На ARM64 использованы функциональные аналоги: SIMD через NEON вместо SSE/AVX, OpenCL через системный framework macOS. Обоснование отступлений приведено в разделе 2 отчёта.

## 📦 Установка зависимостей

```bash
brew install libomp
```

Файлы `stb_image.h` и `stb_image_write.h` должны лежать в корне проекта.

## 🔨 Сборка

```bash
clang++ -std=c++17 -O3 \
  -Xpreprocessor -fopenmp \
  -I$(brew --prefix libomp)/include \
  -L$(brew --prefix libomp)/lib -lomp \
  -framework OpenCL \
  main.cpp -o filters
```

## ▶️ Запуск

```bash
./filters [имя_файла.png]
```

Без аргумента берётся `input.png` по умолчанию.

Пример прогона на нескольких размерах:
```bash
./filters 300x.png
./filters 500x.png
./filters 2400x.png
```

После запуска программа выводит замеры всех 12 функций для указанной картинки и сохраняет результат последнего применённого фильтра в `output.png`.

## 🔍 Ключевые технические решения

**Инверсия SIMD** использует XOR с 0xFF (`veorq_u8`) с маской для сохранения альфа-канала. 16 байтов за такт.

**Собель SIMD** работает в 16-битной арифметике через `vmovl_u8`/`vaddq_s16`. По 8 пикселей за итерацию, корень через `vsqrtq_f32`.

**Медианный SIMD** — ключевая часть работы. Стандартная сортировка содержит ветвления (`if a > b — swap`), несовместимые с SIMD. Решение: оптимальная сортирующая сеть из 19 пар `vminq_u8`/`vmaxq_u8` без единого условного перехода. Каждая операция работает над 16 пикселями, что даёт ускорение до 70× относительно `std::sort`.

**Загрузка пикселей** через `vld4q_u8` — инструкция с автоматическим разделением RGBA на 4 канала.

**OpenCL** инициализируется через системный framework macOS. Kernel-функции лежат в `filters.cl` и JIT-компилируются под GPU при старте.

## 🎯 Основные выводы

1. **Зависимость времени от размера — линейная** O(N) для всех реализаций, что соответствует теоретической сложности алгоритмов.

2. **У каждой технологии своя ниша:**
   - SIMD выигрывает при правильно векторизованных алгоритмах без ветвлений.
   - OpenMP даёт стабильное ускорение ×3–6 практически на любых задачах.
   - OpenCL эффективен только когда работы на пиксель достаточно много, чтобы окупить копирование данных CPU↔GPU.

3. **SIMD на CPU обогнал GPU** на медианном фильтре — потому что у SIMD нет накладных расходов на копирование данных, а сортирующая сеть устранила главное препятствие для векторизации (ветвления).

## 📚 Используемые библиотеки

- [libomp](https://openmp.llvm.org) — поддержка OpenMP в Apple Clang
- ARM NEON Intrinsics — встроенная поддержка SIMD в компиляторе
- Apple OpenCL framework — системный, входит в macOS
